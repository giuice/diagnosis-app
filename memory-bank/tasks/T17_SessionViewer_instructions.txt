# T17_SessionViewer Instructions

## Objective
Create a component for viewing past diagnostic sessions, displaying the full conversation history in a read-only format.

## Context
Parent Implementation Plan: IP4_DiagnosticHistory
This task focuses on implementing the component that displays a selected historical diagnostic session. It will show the full conversation between the user and AI assistant in a read-only view.

## Dependencies
- T11_SessionStore (provides session data)
- T16_HistoryView (for session selection)
- T2_MessageComponents (message rendering foundation)
- shadcn/ui components
- Tailwind CSS

## Steps
1. Create the session viewer component:
   - Create file: `src/components/history/SessionViewer.tsx`
   - Implement main component structure
   - Add container for conversation display
   - Create header with metadata and actions

2. Implement conversation display:
   - Adapt or reuse message components from chat interface
   - Modify for read-only context
   - Ensure proper rendering of both user and AI messages
   - Add visual indicators for read-only state

3. Add session metadata display:
   - Show session title, date, and other metadata
   - Create expandable/collapsible metadata section
   - Add session duration information
   - Implement session tags if applicable

4. Create session actions:
   - Add PDF export button
   - Implement "Resume Conversation" functionality
   - Add delete option with confirmation
   - Create return to list navigation

5. Implement responsive design:
   - Ensure proper layout on different device sizes
   - Create mobile-friendly navigation
   - Add collapsible sections for small screens
   - Implement proper scroll behavior

## Expected Output
A session viewer component that:
- Clearly displays the full conversation history of a selected session
- Shows relevant session metadata
- Provides actions for exporting, resuming, or deleting the session
- Maintains visual consistency with the chat interface
- Clearly indicates its read-only nature

## Notes
- Focus on readability of the conversation history
- Consider adding visual cues to differentiate from active chat
- Ensure accessibility for screen readers
- Add proper keyboard navigation between sessions
- Consider implementing highlight/search within conversation
